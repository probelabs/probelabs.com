<section class="section section-faq">
  <div class="container">
    <div class="section-header">
      <span class="section-label">FAQ</span>
      <h2>Enterprise Questions Answered</h2>
    </div>
    <div class="faq-grid">
      <div class="faq-item">
        <h3>How do you handle massive monorepos (10M+ lines)?</h3>
        <p>Probe is built in Rust with SIMD optimizations specifically for enterprise scale. No pre-indexing required - we search the actual code at query time. Millisecond response times even on the largest codebases because we're not waiting for embeddings or index updates.</p>
      </div>
      <div class="faq-item">
        <h3>Can we run this completely air-gapped?</h3>
        <p>Yes. Zero cloud dependencies. All processing happens locally on your infrastructure. Probe itself requires no network access - pair it with a self-hosted LLM for a fully isolated deployment that meets the strictest security requirements.</p>
      </div>
      <div class="faq-item">
        <h3>What languages do you support?</h3>
        <p>30+ languages including enterprise staples like Java, C#, C++, and Python, plus legacy languages like COBOL and Fortran. Our tree-sitter based parsing understands the actual structure of code, not just text patterns.</p>
      </div>
      <div class="faq-item">
        <h3>How does this integrate with existing tools?</h3>
        <p>MCP protocol for AI agent integration, REST APIs for custom tooling, CI/CD webhooks for automation, and IDE plugins for developer workflows. We fit into your existing infrastructure - you don't rebuild around us.</p>
      </div>
