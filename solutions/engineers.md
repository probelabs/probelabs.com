      <span class="section-label">What Changes</span>
      <h2>Problems This Solves</h2>
      <p class="section-desc">Replace manual context gathering with deterministic, automated workflows.</p>
    </div>
    <div class="pillars-grid">
      <div class="pillar-card">
        <div class="pillar-number">01</div>
        <h3>Slow Onboarding Disappears</h3>
        <p>No more waiting for "the person who knows this system" to be available. Query any codebase with semantic search that understands your code structure, not just string matching.</p>
        <ul class="pillar-benefits">
          <li>AST-aware search finds functions, classes, types</li>
          <li>Cross-repo queries for distributed systems</li>
          <li>Instant answers without interrupting teammates</li>
        </ul>
      </div>
      <div class="pillar-card">
        <div class="pillar-number">02</div>
        <h3>Review Feedback Loops Shrink</h3>
        <p>Stop waiting on obvious review feedback or going through repeated back-and-forth. Deterministic code review catches style violations, security issues, and architectural problems before human review.</p>
        <ul class="pillar-benefits">
          <li>Per-repo rules enforce your standards</li>
          <li>Structured output enables validation</li>
          <li>Auto-fix for common issues</li>
        </ul>
      </div>
      <div class="pillar-card">
        <div class="pillar-number">03</div>
        <h3>Tribal Knowledge Becomes Searchable</h3>
        <p>The knowledge locked in senior engineers' heads becomes accessible. Agent workflows can trace through your actual codebase, pull relevant docs, and explain the "why" behind decisions.</p>
        <ul class="pillar-benefits">
          <li>Architecture diagrams generated from real code</li>
          <li>Dependency analysis across services</li>
          <li>Impact assessment for proposed changes</li>
        </ul>
      </div>

<section class="section section-faq">
  <div class="container">
    <div class="section-header">
      <span class="section-label">FAQ</span>
      <h2>Common Questions from Engineers</h2>
    </div>
    <div class="faq-grid">
      <div class="faq-item">
        <h3>How is this different from GitHub Copilot or Cursor?</h3>
        <p>Copilot and Cursor are great for code completion in a single file. Probe gives you whole-codebase understanding - search across repos, understand dependencies, and run deterministic workflows that integrate with your systems of record.</p>
      </div>
      <div class="faq-item">
        <h3>Does my code leave my machine?</h3>
        <p>No. Probe runs entirely locally. The semantic search index lives on your machine, and you control which LLM provider to use for agent workflows - including self-hosted options.</p>
      </div>
      <div class="faq-item">
        <h3>What languages are supported?</h3>
        <p>Probe supports all major programming languages through tree-sitter parsers: JavaScript/TypeScript, Python, Go, Rust, Java, C/C++, Ruby, and many more. AST-aware search works across all supported languages.</p>
      </div>
      <div class="faq-item">
        <h3>How do I customize the code review rules?</h3>
        <p>Rules are defined as code in your repo. You can specify patterns, validators, and schemas that match your team's standards. The agent produces structured output that gets validated before any comments are posted.</p>
      </div>
